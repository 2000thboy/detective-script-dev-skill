# Outline Format — detective-script-dev

## 强制性格式要求

以下任何一项不满足，大纲将被拒绝：

1. **一句话故事**: 必须存在，简洁概括核心创意，不超过 150 字
2. **故事简介**: 必须存在，200-500 字。需包含：故事链条、人物人设、情节展开方式、真相反转、时代背景、异文化展示
3. **故事细纲**: 必须存在，5000-10000 字。需细化到每章节拍
4. **人物简介**: 必须存在，包含主要角色的性格、动机、关系

## 字数检查规则

| 部分 | 最小字数 | 最大字数 | 不满足时 |
|------|----------|----------|----------|
| 一句话故事 | 10 | 150 | FAIL |
| 故事简介 | 200 | 500 | FAIL |
| 故事细纲 | 5000 | 10000 | FAIL |
| 人物简介 | 200 | 不限 | WARN |

## 禁止事项

- 不得使用其他格式（如单章大纲、角色先行大纲、时间线先行大纲等）
- 不得缺少以上四个核心部分
- 不得将故事简介写成 800-1500 字的过长版本
- 不得将故事细纲写成低于 5000 字的简略版本
- 不得在大纲中混用其他语言（除必要专有名词外）

## 验证命令

```bash
wolf case outline-validate CASE_NAME --outline PATH
```

返回 `PASS` 或 `FAIL` 并列出具体违规项。
